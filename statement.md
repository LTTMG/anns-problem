## ANNS

给定 $d$ 维空间中的 $n$ 个点，以及查询坐标 $q=(q_1,q_2,\cdots, q_d)$，近似最近邻搜索(ANNS)问题旨在从上述 $n$ 个点中查找若干个距查询坐标 $q$ 距离较近的点。

现在给出 $d$ 维空间中的 $n$ 个点，$m$ 个查询坐标，请设计一种算法解决上述ANNS问题。为了避免歧义，当有多个点与查询坐标之间的距离相同时，编号较小的结点优先。

**输入格式**

第一行四个整数 $n,d,q,k$，分别表示点数，空间维数，查询的数量以及要求返回结果的数量；

接下来 $n$ 行，每行 $d$ 个浮点数，表示各点的坐标；

接下来 $m$ 行，每行 $d$ 个浮点数，表示该查询的坐标。

**输出格式**

$m$行，每行 $k$ 个整数，表示查询结果。

**样例输入**

```
5 2 2 2
0 0
1 0
0 1
0 2
1 1
2 1
-1 1
```

**样例输出**

```
5 2
3 1
```

**样例解释**

![](https://github.com/LTTMG/anns-problem/blob/master/res/pic1.png)

在第一次查询中，离查询坐标最近的两个点的编号为2号及5号。

在第二次查询中，1号及4号点与查询坐标距离相同，由于1号点编号更小，故选择1号与3号点。

**限制与约定**

$1\le d\le 128$, $1\le k\le 100$, $k\le n$.

**算法评估**

令算法的正确率为：
$$acc=\frac{\sum_{i=1}^{q}|S_i\cap S_i'|}{q\cdot k}\text{,}$$
其中，$S_i$ 为算法对第 $i$ 个查询的输出，$S_i'$ 为离第 $i$ 个查询坐标最近的 $k$ 个点组成的集合。

你需要在保证算法正确率不小于95\%的前提下尽可能提升算法效率。
